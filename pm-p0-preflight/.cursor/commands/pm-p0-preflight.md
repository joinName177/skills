# P0 澄清预审与草拟回答

按 `.cursor/skills/pm-p0-preflight/SKILL.md` 执行 PMflow P0 预审。

## 执行要求

1. **先读技能**：读取 `pm-p0-preflight/SKILL.md` 全文；题库与审计细则见同目录 `reference.md`。
2. **解析输入**：
   - 若附带「一句话需求」→ 生成 P0 问题 + 草拟回答；
   - 若附带 pm-flow 已输出的 P0 清单 → 审计清单 + 草拟逐条回答；
   - 若 `@` 了 01/02/原型文件 → 优先采信并标注来源。
3. **检索上下文**：在 `原型/模块/` 搜索同模块已有文档与原型。
4. **输出四段**：预审摘要 → P0 表（含草拟答）→ 待确认项 → **可直接粘贴的回复稿**。
5. **禁止**：输出 02 方案、03 功能清单、原型代码；不替代用户对 pm-flow 的正式确认。

## 触发方式

- `/pm-p0-preflight`
- 或自然语言：`预审 P0`、`草拟 P0 答案`、`P0 澄清预审`

## 用法示例

```
/pm-p0-preflight
一句话需求: 表格列头支持快捷筛选
```

```
/pm-p0-preflight
@原型/模块/数据表格/列头快捷筛选/方案/01-需求卡-列头快捷筛选.md
请根据这份 01 草拟 pm-flow P0 回复稿
```

校对草拟稿后，将「可直接粘贴的回复稿」发给 `/pm-flow` 对话即可进入 01/02/03 阶段。
