# pm-p0-preflight

PMflow 的 **P0 澄清预审**技能：在走 `/pm-flow` 正式流程前，把澄清问题问对、草拟答写全，并产出可直接粘贴的回复稿。

## 安装

解压后，将包内 `.cursor` 文件夹**合并**到你的项目根目录（与现有 `.cursor` 并存即可）：

```
你的项目/
└── .cursor/
    ├── skills/pm-p0-preflight/   ← SKILL.md、reference.md、examples.md
    └── commands/pm-p0-preflight.md   ← 可选，斜杠命令入口
```

也可只复制技能目录到个人技能库（全项目可用）：

```
~/.cursor/skills/pm-p0-preflight/
```

## 怎么用（一句话）

**先预审 P0，校对后再进 pm-flow。**

在 Cursor 对话里输入：

```
/pm-p0-preflight
一句话需求: [你的需求]
```

或自然语言：`预审 P0`、`草拟 P0 答案`、`P0 澄清预审`。

## 使用场景

| 场景 | 你怎么说 | 技能帮你做什么 |
|------|----------|----------------|
| **新需求立项** | `一句话需求: 会话列表加通讯录` | 生成 P0 题 + 草拟答，覆盖五维、少漏项 |
| **已有 P0 要填答** | 贴上 pm-flow 输出的 P0 清单 | 审计题目质量 + 逐条草拟回答 |
| **带原型/01 预审** | `@原型/模块/xxx/01-需求卡.md` | 优先采信文档与定稿原型，标注依据 |
| **截图纠错** | 附红框截图 +「这里不对」 | 映射到对应 P0 编号，修正三段不一致 |
| **交互型需求** | 入口、弹框、列表、通讯录等 | 补入口/回退/姿态；场景明确时直接收敛默认 |
| **选项题对不齐** | pm-flow 给了 A/B/C 选项题 | 草拟答按选项映射，不偷换成自由发挥 |
| **进 pm-flow 前省时间** | 先预审再正式流程 | 减少返工；待确认项过多时先补答再开 pm-flow |

**不适合：** 已在写 02/03、只要概念解释、明确要求直接出方案或写代码。

## 典型流程

1. 发一句话需求，或附上 pm-flow 已输出的 P0 清单 / 截图 / `@` 原型或 01 文档  
2. Agent 输出四段：**预审摘要 → P0 表（含草拟答）→ 待确认 → 可粘贴回复稿**  
3. 你校对「待确认」项，改需要改的草拟答  
4. 把「可直接粘贴的回复稿」贴回 `/pm-flow` 对话，进入 01/02/03  

## 它做什么 / 不做什么

| 做 | 不做 |
|----|------|
| 生成/审计 P0 澄清题 | 输出 02 方案、03 功能清单 |
| 结合项目文档与原型草拟答案 | 替代你对 pm-flow 的正式确认 |
| 识别交互入口、回退、姿态等问题 | 写原型代码 |
| 场景已明确时直接收敛交互默认（少甩锅问卷） | 把可推断项机械挂成「待确认」 |

## 示例

```
/pm-p0-preflight
一句话需求: 聊天会话列表增加通讯录，展示群聊和组织成员
```

```
/pm-p0-preflight
@原型/模块/聊天/聊天弹框-浅色玻璃模态.html
请根据定稿原型草拟 P0 回复稿
```

## 文件说明

| 文件 | 用途 |
|------|------|
| `SKILL.md` | 主技能（Agent 必读） |
| `reference.md` | P0 题库、审计细则、交互写法 |
| `examples.md` | 正反例样例 |
| `commands/pm-p0-preflight.md` | Cursor 斜杠命令（可选） |

## 与 pm-flow 的关系

- **pm-p0-preflight** = 预审与草拟（本包）  
- **pm-flow** = 正式产品流程（需项目内 `product-requirement-flow` 规则 + `product-manager` 技能）  

本技能可独立使用；与 pm-flow 联用时体验最佳。
