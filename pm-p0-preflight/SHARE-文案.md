# pm-p0-preflight 分享文案

## 飞书 / 微信群（3 行极简）

**pm-p0-preflight**：PMflow 前置技能，一句话需求先预审 P0，自动草拟澄清答案。  
用法：`/pm-p0-preflight` + 一句话需求 → 校对 → 回复稿贴给 `/pm-flow`。  
适合：新需求立项、P0 清单太长、带截图/原型要预审；不出方案、不写原型。

---

## 稍长版（群公告 / 文档开头）

### 是什么

在走 `/pm-flow` 正式出 01/02/03 之前，用 **pm-p0-preflight** 先把 P0 澄清问题「问对、答全、可粘贴」。

### 怎么用

```
/pm-p0-preflight
一句话需求: [你的需求]
```

输出：预审摘要 → P0 表（含草拟答）→ 待确认项 → **可直接粘贴的回复稿**。  
校对后，把回复稿贴回 `/pm-flow`，再进入需求卡 / 方案 / 功能清单。

### 安装

解压 zip，将包内 `.cursor` 合并到项目根目录。

## 使用场景（何时用）

| 场景 | 你怎么说 | 技能帮你做什么 |
|------|----------|----------------|
| **新需求立项** | `一句话需求: 会话列表加通讯录` | 生成 P0 题 + 草拟答，少漏维度 |
| **已有 P0 要填答** | 贴上 pm-flow 输出的 P0 清单 | 审计题目质量 + 逐条草拟回答 |
| **带原型/01 预审** | `@原型/模块/xxx/01-需求卡.md` | 优先采信文档，标注依据 |
| **截图纠错** | 附红框截图 +「这里不对」 | 映射到对应 P0 编号，修正不一致 |
| **交互型需求** | 入口、弹框、列表、通讯录等 | 补问入口/回退/姿态；场景明确时直接收敛默认 |
| **选项题对不齐** | pm-flow 给了 A/B/C 选项题 | 草拟答按选项映射，不偷换成自由发挥 |
| **进 pm-flow 前省时间** | 先预审再正式流程 | 减少返工，待确认项 ≤3 再开 pm-flow 更稳 |

### 不适合用的场景

- 已经在写 02 方案 / 03 功能清单
- 只要概念解释，不要立项
- 明确说「直接出方案/写代码」

## 与 pm-flow 的分工

```
一句话需求
    ↓
/pm-p0-preflight  ← 预审、草拟、可粘贴回复稿（本技能）
    ↓ 你校对
/pm-flow          ← 01 需求卡 → 02 方案 → 03 功能清单 → 原型
```
